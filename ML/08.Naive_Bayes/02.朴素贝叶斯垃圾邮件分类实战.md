# 02. 朴素贝叶斯垃圾邮件分类实战（从零实现）

这一节把朴素贝叶斯从“公式”落到“能跑的代码”：用 `emails.csv` 做垃圾邮件（spam）分类。

---

## 一、核心公式（只保留你需要用到的）

贝叶斯定理：

`P(A|B) = P(B|A) * P(A) / P(B)`

在垃圾邮件场景中：

- `A`：类别（`spam` 或 `ham`）
- `B`：邮件的单词集合（特征）
- `P(A)`：先验（不看内容时的类别比例）
- `P(B|A)`：似然（在某类别下看到这些单词的概率）

朴素贝叶斯的“朴素”假设（条件独立）：

`P(w1, w2, ..., wn | A) ≈ Π P(wi | A)`

所以分类时常用比较：

- `score(spam) = P(spam) * Π P(wi | spam)`
- `score(ham) = P(ham) * Π P(wi | ham)`

最后：

`P(spam | words) = score(spam) / (score(spam) + score(ham))`

## 二、实现流程（从数据到预测）

### 1) 数据预处理：把文本变成“去重词集合”

目标：只关心某个词“出现/不出现”，不关心次数（与本章的图示直觉一致）。

```python
import pandas as pd

def process_email(text: str) -> set[str]:
    text = (text or "").lower()
    return set(text.split())

emails = pd.read_csv("emails.csv")  # 需要有 text, spam 两列
emails["words"] = emails["text"].apply(process_email)
```

### 2) 先验概率：`P(spam)` 与 `P(ham)`

```python
total = len(emails)
num_spam = int(emails["spam"].sum())
num_ham = total - num_spam

p_spam = num_spam / total
p_ham = num_ham / total
```

直觉检查：如果一封邮件都是未见过的词（信息≈0），预测就会回到先验。

### 3) 统计词在两类邮件中的出现次数（拉普拉斯平滑）

不做平滑会出现“某个词在某类里从没出现过 → 概率 0 → 乘积直接归零”的问题。  
最常见做法：**拉普拉斯平滑**（计数从 1 开始）。

```python
from collections import defaultdict

# model[word] = {"spam": count, "ham": count}
model = defaultdict(lambda: {"spam": 1, "ham": 1})

for _, row in emails.iterrows():
    is_spam = int(row["spam"]) == 1
    for w in row["words"]:
        if is_spam:
            model[w]["spam"] += 1
        else:
            model[w]["ham"] += 1
```

### 4) 预测：建议用 log-space（更稳，不用“乘以 total 防下溢”）

概率连乘很容易下溢（变成 0）。更稳的写法是取对数，把“乘积”变成“求和”：

```python
import math

def predict_naive_bayes(text: str) -> float:
    words = process_email(text)

    # log score
    log_spam = math.log(p_spam)
    log_ham = math.log(p_ham)

    for w in words:
        if w not in model:
            continue

        # 用“出现次数 / 类别样本数”做一个简单的条件概率估计
        # （入门版：把它当成书中“频率当概率”的实现）
        log_spam += math.log(model[w]["spam"] / num_spam)
        log_ham += math.log(model[w]["ham"] / num_ham)

    # 把 log score 还原成概率（两类 softmax）
    m = max(log_spam, log_ham)
    spam_score = math.exp(log_spam - m)
    ham_score = math.exp(log_ham - m)
    return spam_score / (spam_score + ham_score)
```

### 5) 快速测试（与书中直觉一致）

```python
tests = [
    "Hi mom how are you",
    "meet me at the lobby of the hotel at nine am",
    "buy cheap lottery easy money now",
    "asdfgh",
]

for t in tests:
    print(t, "->", predict_naive_bayes(t))
```

## 三、最容易踩的坑（你可以当 checklist）

- **忘记平滑**：某词在某类没出现过 → 概率 0 → 直接把整封邮件判死
- **不统一大小写**：`Lottery` 和 `lottery` 被当成两个词
- **不去重（如果你走的是“出现/不出现”版本）**：会把重复次数当成额外证据，和本章图示直觉不一致
- **直接概率连乘**：易下溢；推荐用 log-space
