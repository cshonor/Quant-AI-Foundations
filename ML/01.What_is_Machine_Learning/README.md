# 第 1 章笔记：什么是机器学习

## 本章笔记整理

### 精简版

- 人类决策有两类：规则推理、经验判断。  
- 机器学习对应“经验判断”，其中经验在计算机里就是数据。  
- 当计算机通过数据学习并决策时，就是在进行机器学习。  
- 对复杂任务（如图像识别），硬编码规则难度高，数据驱动方法更有效。  

### 详细版

#### 核心概念

- **经验 = 数据**：机器学习不是人工穷举规则，而是让模型从数据中归纳规律。  
- **范式变化**：从“程序 + 数据 -> 结果”逐步转向“数据 + 学习算法 -> 模型 -> 结果”。  

#### 与传统编程对比

- 传统编程：人先定义规则，机器执行规则。  
- 机器学习：人提供样本，模型自动学习决策边界。  
- 当任务特征难以完全形式化时（如视觉、语音、自然语言），机器学习通常更可行。  

#### 示例理解（苹果识别）

- 手写规则法：需定义颜色、形状、大小、纹理、光照变化等大量条件，容易遗漏。  
- 数据学习法：通过大量标注样本学习“苹果”特征分布，泛化到新样本。  

#### 一句话总结

> 机器学习的本质是让计算机从数据中积累经验，并用这些经验进行预测和决策。

### 术语补充：模型和算法

#### 核心定义

- **模型（model）**：一组用于表示数据规律的规则，主要用于预测。  
- **算法（algorithm）**：为解决问题而执行的一组步骤，在机器学习里主要用于构建模型。  

#### 关系与区别

- 模型是“学出来的结果”，用于对新数据做判断或预测。  
- 算法是“学习的方法”，决定如何从数据中得到模型。  
- 可记为：**算法训练模型，模型负责预测**。  

#### 书中这一节想表达的重点

- 机器学习先建立术语体系，避免后续概念混淆。  
- “模型”和“算法”常被混用，但职责不同：一个偏结果，一个偏过程。  
- 实践中应区分两件事：  
  1) 我在用什么算法训练；  
  2) 我最终得到的模型表现如何。  

#### 一句话总结

> 模型是对数据规律的可用表达，算法是产生这种表达的步骤与方法。

### 示例 1：烦人的邮件朋友（垃圾邮件判断）

#### 场景

- Bob 经常发来邮件，其中有不少是促销或垃圾内容。  
- 问题是：在不先读完整邮件的情况下，能否先判断它更可能是垃圾邮件还是非垃圾邮件。  

#### 从数据到模型

- 已知历史数据：Bob 过去 10 封邮件中，6 封是垃圾邮件，4 封是非垃圾邮件。  
- 基于这组经验数据，可建立一个简单模型：  
  **Bob 的邮件约有 60% 概率是垃圾邮件。**  

#### 预测思路

- 当新邮件到来时，先按模型给出概率判断：  
  - 垃圾邮件概率：60%  
  - 非垃圾邮件概率：40%  
- 因为 60% > 40%，当前预测倾向于“垃圾邮件”。  

#### 这节的重点

- 机器学习不要求规则绝对真实，而是寻找“基于当前数据的最好可用规则”。  
- 模型会随数据更新：当观察到更多邮件后，60%/40% 这个比例可以被重新估计。  

#### 一句话总结

> 用历史频率构建模型，再用概率大小做预测，是机器学习最直观的入门方式。

### 示例 2：季节性烦人的电子邮件

#### 场景升级

- 在示例 1 中，我们只根据总体频率判断垃圾邮件概率。  
- 这节引入了新信息：**邮件发送时间（星期几）**，尝试寻找更有区分度的模式。  

#### 观察到的模式

- 根据样本数据，Bob 在**工作日**发送的邮件多数是非垃圾邮件。  
- Bob 在**周末**发送的邮件多数是垃圾邮件。  
- 说明“时间特征”能够帮助我们更好地区分两类邮件。  

#### 新模型（模型 2）

- 规则可写成：  
  - 工作日邮件 -> 倾向非垃圾邮件  
  - 周末邮件 -> 倾向垃圾邮件  
- 相比只用总比例（60%/40%）的模型，这个模型更细化，也更贴近数据结构。  

#### 预测示例

- 如果今天是周日，收到 Bob 的新邮件，可高置信度判断为垃圾邮件。  
- 这类判断不需要先读完整内容，可先做自动分拣，再决定是否查看。  

#### 本节要点

- 机器学习效果常来自**特征工程**：选到更有信息量的特征，模型就会更准。  
- 模型不是一次定型，而是随着新特征和新数据不断迭代优化。  

#### 一句话总结

> 从“只看总体频率”到“按时间特征分组建模”，体现了机器学习从粗略到精细的改进路径。

### 示例 3：事情变得复杂了（引入邮件大小特征）

#### 为什么要升级模型

- 按“工作日/周末”判断在大多数情况下有效，但会出现误判。  
- 说明单一特征不足以覆盖所有场景，需要继续寻找更有解释力的特征。  

#### 新观察：邮件大小与类别有关

- 从样本可见：较大的邮件（如 16KB、18KB、20KB、25KB）更多是垃圾邮件。  
- 较小的邮件（如 1KB、2KB、3KB、5KB）更多是非垃圾邮件。  
- 合理解释：垃圾邮件常带附件或大量内容，体积通常更大。  

#### 新规则（模型 3）

- 可建立阈值模型：  
  - 邮件大小 `>= 10KB` -> 倾向垃圾邮件  
  - 邮件大小 `< 10KB` -> 倾向非垃圾邮件  

#### 预测示例

- 若新邮件大小为 19KB，根据模型 3，可预测其更可能是垃圾邮件。  
- 这说明增加有效特征后，模型可在复杂场景中给出更稳健的判断。  

#### 本节要点

- 机器学习是不断迭代的过程：发现误差 -> 增加特征 -> 更新模型。  
- 从“时间特征”到“大小特征”，体现了模型逐步细化与性能提升。  

#### 一句话总结

> 当旧模型解释不了新情况时，引入更有信息量的特征并重建规则，是机器学习改进效果的关键路径。

### 概念补充：什么是特征（feature）

#### 本段核心

- 前面的预测之所以可行，是因为我们使用了可度量的信息，如：  
  - 星期几（时间信息）  
  - 邮件大小（数值信息）  
- 这些可用于区分样本类别的信息，就是“特征”。  

#### 定义

- **特征（feature）**：用于描述数据对象、并可被模型用于预测的属性。  
- 在本书语境中，特征是机器学习里最关键的概念之一。  

#### 在邮件示例中的理解

- 目标：判断邮件是垃圾邮件还是非垃圾邮件。  
- 可用特征示例：  
  - 发送日期（工作日/周末）  
  - 邮件大小（KB）  
- 未来还可扩展更多特征（如关键词、发件频率、附件类型等）以提升判断效果。  

#### 本节要点

- 模型性能不仅取决于算法，也高度依赖特征质量。  
- 选对特征，往往比单纯调参更能显著提升预测效果。  

#### 一句话总结

> 机器学习是在“用特征描述世界”，再让模型基于这些特征做预测。

### 示例 4：更多特征，更多模型

#### 场景升级

- 前面已有两个有效规则：  
  - 按发送时间（工作日/周末）判断  
  - 按邮件大小（是否超过 10KB）判断  
- 这节进一步思考：能否把多个特征组合起来，构建更强的模型。  

#### 模型 4：并联规则（OR）

- 规则：若邮件大小 `> 10KB` **或** 在周末发送，则判为垃圾邮件；否则判为非垃圾邮件。  
- 含义：只要命中任意一个“可疑条件”，就倾向垃圾邮件。  

#### 模型 5：分段阈值规则

- 规则：  
  - 工作日发送时，大小需 `> 15KB` 才判垃圾邮件  
  - 周末发送时，大小需 `> 5KB` 就判垃圾邮件  
  - 否则判为非垃圾邮件  
- 含义：不同时间段使用不同阈值，规则更细，表达能力更强。  

#### 模型 6：数值化组合规则

- 将“星期几”编码为数字（周一 0 ... 周日 6），并与邮件大小相加。  
- 若 `(星期几编码 + 邮件大小KB) >= 12` 判为垃圾邮件，否则判为非垃圾邮件。  
- 含义：把多个特征合成为一个打分信号，属于更抽象的组合建模方式。  

#### 本节关键结论

- 特征和规则可以不断叠加，模型会变得越来越复杂。  
- 模型越来越多时，核心问题从“能否建模”转为“**哪个模型最好**”。  
- 这正是后续需要机器学习算法（训练、评估、选择模型）介入的原因。  

#### 一句话总结

> 机器学习不仅是“设计规则”，更是“在多个候选模型中系统地找到更优模型”。

### 机器使用的一些模型示例（记忆-制定-预测）

#### 三步框架回顾

- **记忆**：观察并记录大量数据。  
- **制定**：通过规则或公式构建模型，并检查哪个模型更贴合数据。  
- **预测**：使用模型对新数据进行判断。  

#### 这一节强调的变化

- 与前文流程一致，但“制定”环节更自动化：  
  计算机可以快速尝试大量规则/公式，找到更匹配数据的模式。  
- 特征维度继续扩展，不再只看时间和大小，还可加入文本内容等特征。  

### 模型 7：规则集合模型（逻辑条件组合）

#### 规则示例

- 若邮件有多个拼写错误 -> 垃圾邮件  
- 若附件大小超过 10KB -> 垃圾邮件  
- 若发件人不在联系人列表 -> 垃圾邮件  
- 若包含“购买”“获胜”等词 -> 垃圾邮件  
- 否则 -> 非垃圾邮件  

#### 理解

- 这是典型的“多条件 if-then”规则系统。  
- 可解释性强，便于人工理解和调试，但规则数量会随场景复杂度快速增加。  

### 模型 8：加权打分模型（公式化）

#### 公式思想

- 将多个特征按权重线性组合成一个分数，再与阈值比较。  
- 书中示意属于这类形式：  
  `score = a1 * 特征1 + a2 * 特征2 + ... + b`  
  若 `score > 阈值` 判为垃圾邮件，否则判为非垃圾邮件。  

#### 理解

- 与模型 7 相比，模型 8 更连续、更可计算，便于后续通过算法自动学习参数。  
- 这是从“人工写规则”向“数据驱动参数学习”过渡的关键一步。  

#### 本节要点

- 同一任务可有不同模型表达：规则树式（模型 7）或公式打分式（模型 8）。  
- 模型表达能力增强后，核心问题仍是：如何评估并选择泛化更好的模型。  

#### 一句话总结

> 机器学习可以用规则或公式来建模，而现代方法更偏向让算法自动学习“公式中的参数”。
