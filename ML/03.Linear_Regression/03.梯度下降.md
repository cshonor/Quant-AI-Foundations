# 03. 梯度下降

## 梯度下降：如何通过缓慢下山来减少误差函数

本节说明如何用一类类似**梯度下降**的方法来减小前面的误差。该过程会用到**导数**，但不必精通微积分也能掌握做法：前面学过的**平方技巧**、**绝对技巧**在每次「沿某方向小挪一步」时，背后其实就是在算误差函数在该方向的导数，从而决定线该往哪移。若要看用导数和梯度推导的完整形式，可参考附录 B。

先回顾线性回归的目标：找到**最拟合数据**的那条直线。**误差函数**（或损失函数）就是用来量化「直线与各点距离」的指标；目标是把误差函数**尽量减小**，得到最佳直线。这在很多数学与机器学习问题中称为**最小化函数**——求函数能取到的最小值。**梯度下降**正是最小化函数的一种常用且有效的方法。

这里要最小化的函数，就是模型的**绝对误差**或**平方误差**（即前面介绍的损失）。需要说明：梯度下降**并非总能找到函数的精确最小值**，但能找到非常接近最小值的解；好在实践中，**梯度下降可以快速、有效地找到函数值较低的点**。

### 梯度下降如何运作？下山类比

**梯度下降相当于下山**。设想你站在一座名为「误差山」（Errorest）的山上，目标是从山上下去。山上**大雾弥漫**，你只能看清大约一米内的范围。

策略是：**环顾四周，弄清朝哪个方向迈出一步能下降得最多**；找到该方向后，就朝那个方向迈出一小步。因为这一步是朝着「下降最多」的方向走的，所以通常会下降一点。**多次重复这个过程，直至到达山脚**（希望如此）。见图 3.20、图 3.21。

- **图 3.20**：在 Errorest 山顶，想往山脚去，但看不清有多远。下山的一种做法是：查看所有能往下走的方向，找出**哪个方向最有助于下山**，然后朝该方向迈出一步，就离山脚更近一点。
- **图 3.21**：下山的方法就是**往「下降最大」的方向迈出一小步，并多次重复该过程**。

这一过程有不少细节（例如可能先到达某个「山谷」局部最低点而无法再下）。此处不展开，附录 B 会给出更详细的数学推导及应对方法；本章重点即**梯度下降**本身。其核心步骤可概括为：

1. 从山上的某个地方开始。
2. 找到最好的方向，迈出一小步。
3. 迈出这一小步。
4. 多次重复第 (2) 步和第 (3) 步。

这与 3.3 节在定义**绝对技巧**和**平方技巧**之后给出的线性回归算法是一致的：先定方向、再小步更新、反复进行。

### 误差函数像山谷：图 3.22

**图 3.22** 把**误差函数**画成一座山谷。目标是下到谷底（误差最小）。山谷里的每个点对应一种模型（一条直线）；点的**高度**表示该模型的误差——**坏模型在山顶（大误差），好模型在谷底（小误差）**。我们从任意一点出发，在**梯度**的指引下反复向下降方向迈小步，使误差逐步减小，最终接近最优解。

换句话说：谷底附近的点对应误差小、表现好的模型；山顶附近的点对应误差大、表现差的模型。用梯度下降就是从当前点开始，一次次沿「下降最多」的方向走，直到到达或接近谷底。

用「直线」的语言复述梯度下降的步骤：

1. 从任意一条直线（初始模型）开始。
2. 用**绝对技巧**或**平方技巧**找到使线稍作移动后误差下降最多的方向。
3. 沿该方向把线**稍微移动**一点。
4. 多次重复第 (2)、(3) 步。

实践中常会画误差随迭代的变化、并设定**何时停止**（如损失不再明显下降或达到最大迭代次数）。常用指标有**均方根误差（RMSE）**和**平均绝对误差（MAE）**，便于理解「平均每栋房预测偏差多少」；参见 `02.损失函数.md`。配套代码中的 `rmse` 函数即用于计算 RMSE。（计算时常用「差向量与自身的点积」表示各分量平方和，再开方得 RMSE。）

### 图 3.23 误差曲线与停止准则

**图 3.23** 是某次运行示例的**均方根误差随迭代次数**变化的曲线。可以看到：误差在大约 1000 次迭代后**快速下降**，之后变化就不大了。因此对该模型而言，可能只需跑 1000 或 2000 次迭代就能得到与跑 10000 次相近的结果，不必每次都跑满 10000 次。

误差曲线为「何时停止」提供了依据；具体停下来的时机往往取决于**可用时间和算力**。两种常见的停止准则为：

1. **损失（或误差）达到事先设定的目标值**时停止。
2. **损失在连续若干迭代周期内不再明显下降**时停止。

### 我们是一次使用一个点还是同时使用多个点？随机梯度下降与批量梯度下降

在 3.3 节中，线性回归是通过「每次选一个点、把线朝该点挪近」来训练的；在 3.4 节中，则先对**整个数据集**算误差（绝对误差或平方误差），再用梯度下降减小该误差来训练。为什么 3.4 里是对整份数据算误差？

实际上，训练时既可以**每次只用一个点**更新，也可以**每次用整份数据**更新。当数据量很大时，这两种方式都可能很耗算力。折中的做法是**小批量学习（mini-batch）**：把数据分成很多个小的「小批量」，每轮迭代选一个小批量，只在该小批量上算误差并更新模型参数，使该批内的误差下降。

按「每次迭代用多少数据」可区分三种梯度下降：

| 方式             | 名称                         |
|------------------|------------------------------|
| 每次用**一个点** | **随机梯度下降**（SGD）      |
| 每次用**一小批点** | **小批量梯度下降**         |
| 每次用**整份数据** | **批量梯度下降**           |

更详细的推导与实现可参考附录 B。

## 核心概念与算法步骤

### 核心问题

损失函数给出了「好/坏」的度量，但如何让计算机**自动找到**使损失最小的权重和偏差？梯度下降（Gradient Descent）就是常用的求解算法。

### 直观理解：下山

- 想象站在山坡上，目标是走到最低谷（损失最小）。
- **梯度**：指向上坡方向的向量；反方向就是下坡。
- **梯度下降**：每次沿梯度反方向迈一小步，逐步靠近谷底。

### 基本步骤

1. **初始化**：随机或按规则设定初始的 w、b。
2. **计算梯度**：求损失函数对 w 和 b 的偏导数。
3. **更新参数**：
   - w ← w − α · (∂L/∂w)
   - b ← b − α · (∂L/∂b)
4. **重复**步骤 2、3，直到损失变化很小或达到最大迭代次数。

其中 α 为**学习率**，控制每步迈多大。

### 学习率

- **太小**：收敛慢，需要很多步。
- **太大**：可能跨过最优点，甚至发散。
- 通常需要根据实验调整。

### 在房价模型中

- 损失 L 由 MSE 定义，是 w 和 b 的函数。
- 梯度下降不断更新 w、b，使 L 逐渐减小。
- 最终得到的 w、b 对应「最接近所有点」的那条直线。

### 一句话总结

> 梯度下降通过沿损失函数梯度的反方向更新参数，逐步逼近使损失最小的解。
