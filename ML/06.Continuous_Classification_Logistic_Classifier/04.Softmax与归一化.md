# 04. Softmax 与归一化详解

这一篇解决两个常被混在一起的概念：

- **Softmax**：把多分类的“原始分数（logits）”变成**合法概率分布**（每一项都在 (0,1)，且总和为 1）
- **归一化（Normalization）**：更广义的“把数据缩放到统一尺度/合法范围”，Softmax 里用到的是其中一种：**概率归一化**

> 记号约定：
>
> - `a_i`：第 `i` 类的原始分数（logit）
> - `p_i`：第 `i` 类的预测概率
> - `exp`：自然指数（书中也常用 `e^x` 写）

---

## 一、Softmax 函数

### 1) 核心定义（它在多分类里干什么）

Softmax（也叫**归一化指数**）是二分类里 **sigmoid** 思路的自然推广：

- 输入：一组实数分数 `a1, a2, ..., aK`（`K` 个类别）
- 输出：一组概率 `p1, p2, ..., pK`，满足：
  - 每个 `pi` 都在 **(0,1)**
  - `p1 + p2 + ... + pK = 1`

因此 softmax 常被当作多分类模型的**标准输出层**：它不仅告诉你“更像哪一类”，还告诉你**在全部类别之间如何分配置信度**。

### 2) 数学公式（用代码行写，避免 LaTeX）

对第 `i` 类：

`pi = exp(ai) / (exp(a1) + exp(a2) + ... + exp(aK))`

直觉分三步：

1. 对每个分数做 `exp`，得到一堆**正数权重**
2. 把这些权重加起来得到**总权重**
3. 每个类的概率 = **自己的权重 / 总权重**（这就是“百分比化/归一化”）

### 3) 计算示例（狗 / 猫 / 鸟，三分类）

三个原始分数：

- 狗：`a1 = 3`
- 猫：`a2 = 2`
- 鸟：`a3 = -1`

#### 步骤 1：指数化（全部变正，负分数也能参与）

- `exp(3) ≈ 20.085`
- `exp(2) ≈ 7.389`
- `exp(-1) ≈ 0.368`

#### 步骤 2：求和（总权重）

`S ≈ 20.085 + 7.389 + 0.368 = 27.842`

#### 步骤 3：归一化（每个权重除以总和）

- `p1 ≈ 20.085 / 27.842 ≈ 0.721`
- `p2 ≈ 7.389 / 27.842 ≈ 0.265`
- `p3 ≈ 0.368 / 27.842 ≈ 0.013`

验算：`0.721 + 0.265 + 0.013 = 0.999`（四舍五入误差；精确计算应为 1）

### 4) 核心性质（建议你按条背）

1. **输出恒正**：`exp(ai) > 0`，所以 softmax 输出不会出现负数概率这种“非法值”。
2. **输出和为 1**：天然是概率分布，可直接解释“属于各类的占比”。
3. **保留排序信息**：`ai` 越大，对应的 `pi` 越大（单调性直觉）。
4. **处处可导**：适合配合梯度型训练（细节推导后面再补也完全来得及）。
5. **与 sigmoid 的关系（重要）**：二分类时 softmax 与 sigmoid 是同一套思想的两种写法；多分类时 softmax 是更“标准”的整体归一化版本。

### 5) 工程提醒：数值稳定性（你知道有这回事就行）

当 `ai` 很大时，`exp(ai)` 可能溢出；常见做法是先把所有 logits 减去 `max(a1..aK)` 再做 softmax（数学上等价，但更稳）。实现细节留到写代码章节再展开。

## 二、归一化（Normalization）详解

### 1) 核心定义（广义）

归一化是一类把数据映射到**更统一、更好训练/更好解释**的范围的操作。

在 softmax 里，我们关心的是其中一种：**概率归一化**——把一组非负权重变成总和为 1 的概率。

### 2) Softmax 里的归一化到底在做什么？

你可以把它理解成三步话的“百分比化”：

1. 先把每个类的分数变成“非负权重”（softmax 用 `exp`）
2. 再算总权重
3. 最后每个类的概率 = 自己的权重占总权重的比例

没有“除以总和”这一步，你就很难保证输出天然满足概率公理。

### 3) 机器学习里其它常见的归一化/标准化（目标不同）

它们都叫“归一化/缩放”，但**目的不一定是为了输出概率**：

| 类型 | 常见写法（直觉） | 做什么 | 常见用途 |
|---|---|---|---|
| Min-Max 缩放 | `x' = (x - xmin) / (xmax - xmin)` | 把特征大致压到 `[0,1]`（或固定范围） | 让不同特征尺度更接近，训练更稳 |
| Z-score 标准化 | `x' = (x - mean) / std` | 让特征大致“中心化 + 统一波动尺度” | 线性模型/梯度类训练里很常见 |
| Softmax 概率归一化 | `pi = exp(ai) / Σ exp(aj)` | 把 logits 变成**总和为 1**的概率 | 多分类输出层 |

### 4) 关键澄清：归一化 ≠ 正则化（Regularization）

| 概念 | 主要动的是谁 | 想解决什么 |
|---|---|---|
| 归一化 | 往往是**数据/输出**的尺度 | 让数值更合理、更好优化、或得到合法概率 |
| 正则化 | **模型参数**（权重等） | 抑制过复杂、降低过拟合风险（例如对权重大小加惩罚项） |

一句话：**归一化更像“把数摆好”**；**正则化更像“别学太野”**。

---

## 三、Softmax 与归一化的关系（收尾总结）

1. **Softmax = 指数化 + 概率归一化**：先把 logits 变成非负权重，再除以总和得到概率。
2. **归一化是 softmax 能成为概率输出的关键步骤**：没有“总和为 1”的约束，就很难把多分类输出解释成概率分布。
3. **softmax vs sigmoid（多输出时别混）**：
   - **sigmoid**：常用于“每个类各自一个 0~1 输出”（多个 sigmoid 时，输出**不必**相加为 1）
   - **softmax**：对**一整组 logits**做一次竞争式分配，输出**必须**相加为 1（多分类更标准）

---

## 四、极简背诵版（写在小抄上）

> Softmax 把多分类 logits 变成**合法概率**：先 `exp` 变正权重，再除以总和做归一化；总和为 1 是它的灵魂。  
> “归一化”在机器学习里还有很多种，但**概率归一化**与 **Min-Max / Z-score** 的目标不同。  
> **归一化 ≠ 正则化**：一个主要摆数据的尺度，一个主要约束模型复杂度。
