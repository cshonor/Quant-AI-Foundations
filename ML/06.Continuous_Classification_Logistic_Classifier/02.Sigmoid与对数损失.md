# 02. Sigmoid 与对数损失（Log Loss）详解

这一份笔记把两件事讲透：

- **Sigmoid**：为什么它能把“任意实数分数”变成 **0～1 的概率/倾向**
- **Log Loss（对数损失）**：为什么它适合作为逻辑回归的训练目标，以及它和 sigmoid 怎么配合

> 记号约定（全书自洽最重要）：
>
> - `y ∈ {0,1}`：真实标签（本书例子里常把 **1** 当作正类，比如“高兴”）
> - `ŷ ∈ (0,1)`：模型预测“正类（y=1）”的概率（由 sigmoid 输出）
> - `ln`：自然对数（书中也常用 `log` 写自然对数，注意上下文）

---

## 一、Sigmoid 函数

### 1) 它是什么？（定义）

Sigmoid（常记作 `σ`）是一个把实数“压”进开区间 **(0,1)** 的函数，最常见的写法是：

`σ(z) = 1 / (1 + exp(-z))`

在逻辑回归/逻辑分类器里，`z` 通常不是随便一个数，而是线性打分：

`z = a·x1 + b·x2 + ... + c`

也就是你先做“线性部分”，再送进 sigmoid，得到概率式输出 `ŷ = σ(z)`。

### 2) 你最该记住的 6 条性质（考试/面试够用了）

1. **值域是 (0,1)**：输出可以无限接近 0 或 1，但一般**不会正好等于** 0 或 1（这一点和“概率模型输出”很合拍）。
2. **定义域是全体实数**：`z` 可正可负、可大可小，没有“输入不合法”的问题。
3. **严格单调递增**：`z` 越大，`σ(z)` 越大；更符合“分数越高越像正类”的直觉。
4. **处处连续、可导**：曲线是光滑 S 形；这也是它能和“梯度型训练”绑定的关键原因。
5. **对称中心在 (0, 0.5)**：`σ(0) = 0.5`，常被当作**决策阈值**（再配合硬分类规则：≥0.5 判正类）。
6. **导数形式很省事**：  
   `σ'(z) = σ(z) · (1 - σ(z))`  
   这意味着：用一次前向算出 `σ(z)`，导数也能顺手写出来，训练实现很省。

### 3) 它解决什么问题？（和阶跃函数对比）

感知器里常用的 **阶跃函数**很“硬”：输出几乎只有 0/1，阈值处不可导、也很难表达“把握有多大”。

Sigmoid 的核心作用可以一句话概括：

> **把线性分数变成可训练的概率输出**，让模型既能分类，又能输出“置信度/倾向”。

## 二、对数损失（Log Loss）

### 1) 它是什么？（定义）

对数损失用来衡量：模型给出的概率 `ŷ`，到底有多贴合真实标签 `y`。

**单个样本**常见的一种写法是：

`loss = -( y·ln(ŷ) + (1-y)·ln(1-ŷ) )`

它本质是一个“开关公式”：

- 当 `y = 1`：第二项消失，变成 `loss = -ln(ŷ)`
- 当 `y = 0`：第一项消失，变成 `loss = -ln(1-ŷ)`

**整个数据集**（`n` 个样本）的平均损失常写成：

`avg_loss = -(1/n) * Σ( y_i·ln(ŷ_i) + (1-y_i)·ln(1-ŷ_i) )`

训练目标就是：**让 `avg_loss` 变小**（等价于让模型更给真实标签“面子”，把概率质量放到正确标签上）。

### 2) 你最该记住的性质（抓住直觉，别背套话）

1. **惩罚“自信但错了”特别狠**：  
   - `y=1` 但 `ŷ` 很接近 0 → `-ln(ŷ)` 会非常大  
   - `y=0` 但 `ŷ` 很接近 1 → `-ln(1-ŷ)` 会非常大  
   这正是分类训练里很想要的行为：**错得越离谱、越自信，代价越大**。

2. **预测越贴近真实标签，损失越小**：  
   `ŷ` 越接近 `y`（在 `y=1` 时接近 1，在 `y=0` 时接近 0），`loss` 越接近 0。

3. **和 sigmoid 很“配套”**：sigmoid 提供 `(0,1)` 的概率输入；log loss 在这个区间里好用、可导，适合梯度型优化。

4. **把“连乘概率”变成“求和 log”更稳**：  
   独立样本的联合概率常是乘积；取对数后变求和，数值更稳定（避免特别多的小数连乘导致下溢）。

### 3) 它和“交叉熵”是什么关系？

你会在很多框架里看到 **cross entropy** 这个词。入门阶段可以先把 log loss 理解成：

> **用概率语言衡量“预测分布”和“真实标签（0/1）”之间的距离**。

更严格的概率分布版本后面再展开也完全来得及。

## 三、Sigmoid + Log Loss：二者如何拼成逻辑回归的“发动机”

一句话串起来：

- **Sigmoid**：负责把线性分数 `z` 变成 `ŷ`（概率输出）
- **Log Loss**：负责告诉参数“这次预测有多糟”，从而指导更新

所以逻辑回归不是“随便套一个曲线”，而是两件东西**天然配套**：

- 输出在 (0,1)，损失在概率上说话
- 训练时能用“看梯度、走一步”的系统化方法把参数越调越好

## 五、易混淆点澄清：Sigmoid vs 哈希（Hash）

有些人会直觉地把 sigmoid 想成“把任何东西压缩成一个固定范围的数”，于是容易和 **哈希函数**混在一起。它们确实在“表面”上有一点像，但数学目标完全不同。

### 1) 哪里看起来有点像？

- **都把输入映射到有限范围**：sigmoid 把实数压到 **(0,1)**；哈希通常把任意长度输入映射到**固定长度**的摘要（看起来像“落到有限集合里”）。
- **输入变了，输出往往会变**：两者都对输入变化敏感。

### 2) 本质差别是什么？（一句话版）

- **Sigmoid**：要的是**平滑、单调、可导**——输出连续变化，方便优化（梯度思想）。
- **哈希（尤其是密码学哈希）**：要的是**看起来像随机、难以预测/难以逆向**（以及抗碰撞等安全性质）——它不是为了“训练模型”。

### 3) 更具体一点：别把“不可预测”当成同一种东西

- **Sigmoid 的“变化”是平滑可控的**：`z` 稍微变大一点，`σ(z)` 也只会**平滑地**变大一点；而且整体 **单调递增**。
- **哈希的“变化”通常是剧烈且不保留远近关系的**：输入改一点点，输出摘要可能**完全不像**（这是设计目标之一），也不存在“离输入近 ⇒ 输出也近”这种几何直觉。

### 4) 对训练/优化意味着什么？

- **Sigmoid**：处处可导（在机器学习语境里非常好用），因此能和对数损失、梯度型更新配合。
- **哈希**：不是为了求导优化设计的；把它当激活函数去训练，通常既不自然也不合适。

### 5) 核心结论（背诵用）

- **Sigmoid**：服务 **可学习模型**（概率输出 + 可优化）
- **哈希**：服务 **标识/校验/摘要**（离散摘要 + 安全性质）

二者“都会压缩”，但压缩的目的与数学性质不是一类问题。
