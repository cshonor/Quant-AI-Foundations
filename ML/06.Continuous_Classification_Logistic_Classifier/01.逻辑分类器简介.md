# 01. 逻辑分类器简介

这一章回答一个很现实的问题：分类不只是“对/错”，我们还常常想知道**把握有多大**。

下面我按你教材里的编号顺序，把 **图 6.1 → 图 6.2 → 图 6.3 → 表 6.1 → 表 6.2 → 图 6.4 → 表 6.5** 串成一条新手也能顺着读完的链路（配图都在本文件夹里）。

> 小提示：第 5 章已经把 `score = w·x + b`、分界线、以及“`score` 大小和离边界远近”的几何直觉讲得很细了；本章重点放在：**如何把 score 变成 0～1 的概率式输出**，以及**为什么训练要用对数损失**。

---

## 图 6.1：同样的分界线，输出可以是“硬 0/1”，也可以是“软概率”

图 6.1 左右两幅图用的是**同一条分界线**、同一批点，差别只在输出形式：

- **左：感知器（离散）**  
  分界线一侧输出 **1**，另一侧输出 **0**。离分界线远不远，输出都还是 0 或 1——**没有“把握程度”**。

- **右：逻辑回归 / 逻辑分类器（连续）**  
  输出变成 **0 到 1 之间的小数**。通常把它理解成：**模型觉得样本属于正类（比如更偏“高兴”）的概率或倾向**。

![图 6.1（左：离散感知器；右：连续逻辑回归）](./imagine6.1.png)

读图时你只要抓一个规律：**点离分界线越近，输出越接近 0.5（更不确定）；离分界线越远，输出越接近 0 或 1（更确定）**。

> 备注：有些教材配图下方的文字说明，可能把“0/1 各自对应哪一类”写得很简略；读图时以坐标分区上的文字（高兴/悲伤）和点的标注为准，最重要的是**全书符号自洽**。

---

## 图 6.2：sigmoid 把整条数轴“压进”(0, 1)

你可以把 `score`（也就是线性部分 `w·x + b`）想成**任意实数**：可正可负、可大可小。

sigmoid 做的事很直观：**把 (-∞, +∞) 映射到 (0, 1)**，所以输出天然适合当“概率/倾向”。

![图 6.2（sigmoid：整条数轴映射到 (0,1)）](./imagine6.2.png)

看图里几个“锚点”会特别好记：

- `score = 0` → 输出落在中间（0.5）：**最不确定**
- `score` 越正 → 输出越靠近 1：**越像正类**
- `score` 越负 → 输出越靠近 0：**越不像正类**

---

## 图 6.3：把阶跃函数换成 sigmoid，到底换了什么？

图 6.3 把两种激活函数摆在一起，差别一眼就能看出来：

- **左：阶跃函数（step，离散）**  
  输出几乎只有两档（0/1），在阈值附近会“跳变”。它是**硬判决**。

- **右：sigmoid（连续）**  
  输出在 0～1 之间连续变化，曲线光滑。它是**软输出**。

![图 6.3（左：阶跃；右：sigmoid）](./imagine6.3.png)

为什么要换成 sigmoid（至少两条新手最该记住的理由）：

- **能输出“把握有多大”**：同一个“判对/判错”的问题，逻辑分类器还能告诉你更偏哪一边、离 0.5 有多远。
- **更利于用“平滑损失”去训练**：step 在跳变处太“硬”，很多基于梯度思想的训练细节会更难处理；sigmoid 这种连续曲线更“好捏形状”。

表 6.1 给了几组输入输出，用来建立直觉：**大正数靠近 1，大负数靠近 0，0 在中间**。

![表 6.1（sigmoid 的一些输入与输出）](./imagine-table6.1.png)

你可以把表里的数当成“手感校准器”：

- 输入从 0 走到 1，输出会从 0.5 明显往上抬
- 输入继续变大，输出会**越来越接近 1**，但通常不会完全等于 1（同理负数侧接近 0）

## 图 6.4：同一条分界线 + 概率输出：为什么还会“判错”？

图 6.4 把表 6.2 的点画到平面上，并给出模型对“正类（高兴）”的预测概率（0～1）。

![图 6.4（表 6.2 的数据集与预测；点 2、4 正确；点 1、3 错误）](./imagine6.4.png)

图里文字说明了一个最常用的“硬分类规则”（本章先固定用它）：

- 预测概率 **≥ 0.5** → 判到“正区”（更偏高兴）
- 预测概率 **< 0.5** → 判到“负区”（更偏悲伤）

你会发现两类“错误”并不一样：

- **错得很自信**：真实标签是 0，但预测概率很高（接近 1）——模型“非常确定但完全错了”。
- **错得不自信**：真实标签是 1，但预测概率很低（接近 0）——模型“更偏另一边”。

这也自然引出下一问题：训练时，我们该用什么样的误差函数，才能让模型**尤其害怕**第一种错误？

---

## 表 6.5：对数损失（log loss）——“越自信越错，罚得越狠”

表 6.5 在算每个点的 **对数损失**。你可以先不用背公式，先记住它的“脾气”：

- 先取出“模型对**真实标签**给出的概率”（表里叫“标签概率”）
- 再对它取 `ln`，并加一个负号（所以叫对数损失）

直觉：

- **标签概率越接近 1**（模型很相信真实标签）→ 损失越小
- **标签概率越接近 0**（模型几乎不相信真实标签）→ 损失会急剧变大

![表 6.5（数据集点的对数损失计算）](./imagine-table6.5.png)

用一句话对齐表里的现象：

- **点 2、点 4**：分类更靠谱/更自信地对上了标签 → **损失更小**
- **点 1、点 3**：分类不对，尤其点 1 属于“很自信但错了” → **损失更大**

这也解释了逻辑分类器与感知机在“误差”层面的关键差别：

- **感知机**：很多时候更像在围绕“分错/分对、错得离边界多远”做文章（第 5 章主线）。
- **逻辑分类器**：常用 **log loss** 这种概率损失，让模型对“自信但错误”付出更高代价，训练往往更稳定、更可控。

## 阈值 0.5（本章默认）

得到 0～1 的预测概率后，如果仍要输出硬分类（开心/难过），通常设阈值 **0.5**：

- 预测概率 ≥ 0.5 → 判为正类
- 预测概率 < 0.5 → 判为负类

0.5 是常见、简便的默认选择；第 7 章会讨论如何调整阈值来优化实际指标（准确率/召回率等）。

## 补充直觉：为什么要“0～1 的分数”？

在现实任务里，我们往往不仅想知道“正/负”，还想知道：

- 这次判断**有多确定**？
- 这是“勉强算正类”，还是“几乎肯定是正类”？

逻辑分类器输出的 0～1，就可以表达这种“确定程度/正类倾向”。例如：

- 0.60：更偏正类，但不是很确定
- 0.95：强烈偏正类，非常确定
