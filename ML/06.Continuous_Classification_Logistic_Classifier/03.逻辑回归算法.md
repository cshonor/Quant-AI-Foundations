# 03. 如何找到一个好的逻辑分类器？逻辑回归算法（6.2）

这一节是第 6 章里把前面内容“落到算法上”的关键段落：前面已经会用 **sigmoid** 把线性分数变成 **0～1 的概率**，也已经会用 **log loss（对数损失）**衡量模型好坏；6.2 要回答的是：

> 我该怎么一步步把 `a,b,c`（权重与偏置）调到更优？

书里给出的答案就是：**逻辑回归（Logistic Regression）算法**——你可以把它理解成“连续版感知器训练”，但更新规则从“对错一跳”变成了“按误差大小连续地挪一点点”。

---

## 一、核心定位（6.2 在讲什么）

6.2 承接三件事：

- **模型形式**：`ŷ = sigmoid(a·x1 + b·x2 + c)`（书中例子常用 `x_aack`、`x_beep` 作为两个特征）
- **优化目标**：让整体 **log loss** 变小（等价于让模型对真实标签给出的概率更高）
- **更新机制**：反复随机看点，用“逻辑技巧”微调参数（本质是 **SGD：随机梯度下降**）

一句话：**逻辑回归 = 用 sigmoid 输出概率 + 用 log loss 当目标 + 用梯度思想迭代更新参数**。

---

## 二、训练流程（你可以按这 3 步记）

### 1) 初始化

给权重 `a,b` 和偏置 `c` 一个起点：

- 常见做法：**随机初始化**（也可能从 0 开始，取决于实现与数据规模）

### 2) 迭代优化（重复很多次）

每一轮通常做这些事：

1. **随机抽一个样本** `(x1, x2, y)`（这里的 `y` 是真实标签，取 0 或 1）
2. 用当前参数算预测概率：  
   `ŷ = sigmoid(a·x1 + b·x2 + c)`
3. 用下面的“逻辑技巧”更新 `a,b,c`（让这次样本上的损失倾向于下降）

> 直觉：这和第 5 章感知器“看一个点就改一点点”的结构很像，只是**更新量不再只靠对错**，而是看 **`y - ŷ` 有多大**。

### 3) 停止条件

实践中会用一些规则让循环停下来，例如：

- 迭代次数到上限（比如你设定重复 `n` 轮）
- log loss 变化很小（“基本收敛”）
- 验证集指标不再提升（更工程化）

---

## 三、“逻辑技巧”到底是什么？（几何直觉 + 更新公式）

### 1) 几何直觉：边界在“远离/靠近”样本点

书里常用一张示意图（你教材里一般标成 **图 6.6**）把更新讲得很像“推分界线”：

- **分对了**：把分类边界往“让点更自信”的方向挪一点——直观上像是**让边界离这个点更远**（点更深地进入正确一侧，预测更靠近 0 或 1）
- **分错了**：把分类边界往“纠错”的方向挪一点——直观上像是**让边界更靠近这个点**（给它机会被翻到正确一侧）

> 重要：这是“新手读图直觉”，真正严格对应到每一次更新，仍要以书里的推导/伪代码为准；但它的确帮助你把 SGD 想成“会动的线”。

### 2) 代数更新：用 `y - ŷ` 当误差信号

对二分类（标签 `y` 为 0/1），一个很常见、也很好记的更新形式是：

- `a' = a + η · (y - ŷ) · x1`
- `b' = b + η · (y - ŷ) · x2`
- `c' = c + η · (y - ŷ)`

其中：

- `ŷ = sigmoid(a·x1 + b·x2 + c)`：当前模型对“正类（y=1）”的预测概率
- `η`：学习率（步长），控制每次改动有多大

你可以用三条“观察”记住它为什么长这样（不必背证明）：

1. **`(y - ŷ)` 是误差方向**：预测偏低就往上拉，预测偏高就往下压  
2. **乘以 `x1/x2`**：特征越大，说明它对这次预测“更有发言权”，权重更值得被多改一点  
3. **乘以 `η`**：把改动限制在合理步长里，避免一次改飞

### 3) 学习率 `η` 怎么取（工程直觉）

没有放之四海皆准的“唯一正确答案”，但新手可以先用区间感：

- **太大**：更新像“猛踩刹车/猛踩油门”，损失可能上下抖，甚至更难收敛  
- **太小**：更新像“挪蚂蚁”，训练很慢，需要更多迭代

实际项目里常常还会配：**衰减学习率、早停、正则化**等（后面章节/工程实践会继续展开）。

这一节的“验收方式”通常很直观：

- 对同一个数据集，反复应用更新后：
  - **`ŷ` 会更靠近真实 `y`**
  - **每个点的 log loss 会下降**，总损失整体往下走
- **图 6.7** 往往会把“边界怎么移动”画出来，让你能把代数更新与几何移动对上号

## 五、伪代码（抓住结构就够）

输入：

- 标签为 0/1 的数据集
- 迭代次数 `n`
- 学习率 `η`

输出：

- 训练好的参数：`a,b,c`

流程：

1. 初始化 `a,b,c`
2. 重复 `n` 次：
   1. 随机选一个样本 `(x1, x2, y)`
   2. 计算 `ŷ = sigmoid(a·x1 + b·x2 + c)`
   3. 更新：
      - `a ← a + η·(y-ŷ)·x1`
      - `b ← b + η·(y-ŷ)·x2`
      - `c ← c + η·(y-ŷ)`
3. 返回 `a,b,c`

## 六、梯度下降的三种常见“工程形态”

你可以把逻辑回归里“每次用一个点更新”理解为 **SGD（随机梯度下降）**。工业界还会用变体：

| 类型 | 每次更新用多少样本 | 特点 |
|---|---|---|
| SGD | 1 个 | 快、噪声大；和本书“随机抽点更新”最像 |
| Mini-batch | 一小批（如 32/64） | 速度/稳定性折中；很常见 |
| Batch | 全量 | 更“稳”、更慢；数据很大时不划算 |

## 八、新手最容易踩的坑（建议你写进笔记本角落）

- **名字叫“回归”，任务是分类**：这里的“回归”更像历史命名，不要把它当成“预测房价那种回归任务”。
- **损失别乱换成 MSE**：在概率输出上直接套回归损失，优化问题可能变得很难搞（不凸/不好训练等），这不是入门阶段该走的弯路。
- **特征量纲会影响更新**：如果不同特征尺度差很多，`x` 大的特征会把更新“带跑”。实际工程里常见做法是标准化/归一化（后面章节或实践里会系统讲）。
