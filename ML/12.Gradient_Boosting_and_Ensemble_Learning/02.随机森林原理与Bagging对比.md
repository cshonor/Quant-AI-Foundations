# 02. 随机森林原理与 Bagging 对比

本节对应《机器学习图解》第 12 章中关于 **Random Forest（随机森林）** 的核心内容。随机森林属于 **Bagging（Bootstrap Aggregating）** 路线：让多个相互独立的弱学习器在不同随机样本上训练，再把结果投票/平均，得到更稳健的整体预测。

---

## 一、随机森林是什么

- **所属流派**：传统机器学习，集成学习中的 **Bagging** 家族。  
- **基本单位**：通常是多棵决策树。  
- **核心目标**：通过“多个不太一样的树”共同投票，降低单棵树的波动，缓解过拟合。  

随机森林的“随机”，主要来自两层：

1. **数据随机**：对训练集做 **Bootstrap 有放回抽样**，得到多个子集。  
2. **特征随机**：每棵树在分裂时只看一部分随机抽到的特征。

## 三、Scikit-Learn 实现

```python
from sklearn.ensemble import RandomForestClassifier

# n_estimators: 树的数量
# max_depth: 每棵树的最大深度
# random_state: 随机种子
random_forest_classifier = RandomForestClassifier(
    random_state=0,
    n_estimators=5,
    max_depth=1,
)

random_forest_classifier.fit(features, labels)
```

这里 `max_depth=1` 表示每棵树都只是一个很浅的弱学习器；但通过集成，整体仍能形成有效边界。

## 四、Bagging vs Boosting

| 维度 | Bagging（随机森林） | Boosting（AdaBoost 等） |
|------|---------------------|-------------------------|
| 训练方式 | 并行、彼此独立 | 串行、后一个依赖前一个 |
| 数据处理 | Bootstrap 抽样，样本权重相同 | 动态调整样本权重 |
| 主要收益 | 降低方差，缓解过拟合 | 降低偏差，逐步纠错 |
| 代表算法 | Random Forest | AdaBoost、Gradient Boosting、XGBoost |

## 五、学习建议

如果你的当前目标是读懂《图解大模型》，那么这一节**不是必学前置**。  
随机森林是结构化数据任务里的经典算法，但和 Transformer / 注意力机制 / LLM 并不是同一技术路线。可以先知道它属于传统机器学习，再把主要精力放到深度学习主线。
