# 05. 正则化

## 4.5 避免过拟合的另一种选择：正则化

除了用测试集、验证集和模型复杂度图来选模型，还可以用**正则化（regularization）**来减轻过拟合。正则化在**不一定单独划出测试集**的情况下也很有用：通过在损失函数里加入对模型复杂度的惩罚，让训练过程**一次**就同时兼顾「拟合得好」和「不要太复杂」，从而降低过拟合风险。

这与 01 中的观察一致：**太简单的模型**容易欠拟合，**太复杂的模型**容易过拟合。之前我们是**训练多个模型**（如不同阶数的多项式），再通过验证/测试选一个；正则化的思路则是**只训练一次**，在目标里同时考虑「表现好」和「复杂度低」，从而在单次训练中达到平衡。

---

## 衡量模型性能与复杂度的类比：屋顶修补（图 4.6）

为了直观理解「性能」与「复杂度」的权衡，书中用**屋顶漏水**来类比（见**图 4.6 欠拟合和过拟合的类比**）。屋顶工 1 带绷带（过度简化→欠拟合），屋顶工 2 用很好的解决方案（如屋顶瓦），屋顶工 3 带钛金属（过度复杂→过拟合）。

**场景**：三栋房子的屋顶都在漏水，需要修补。

**三位屋顶工**（对应三种「模型」），各修一栋：

| 屋顶工   | 修补方式   | 直观评价           | 对应现象   |
|----------|------------|--------------------|------------|
| 屋顶工 1 | 用**绷带** | 过度简化了问题     | **欠拟合** |
| 屋顶工 2 | 用**屋顶瓦** | 直觉上最合适     | **合适拟合** |
| 屋顶工 3 | 用**钛**   | 将问题过度复杂化   | **过拟合** |

绷带太简陋，修不好 → 欠拟合；钛过于昂贵、大材小用 → 过拟合；屋顶瓦刚刚好 → 合适拟合。

---

## 用数值衡量：漏水量（毫升）

若要**用数据做决策**，需要可量化的指标。这里用「修补后仍泄漏的水量（毫升）」来衡量修得好不好：

- **屋顶工 1**：1000 毫升（漏得很多）→ 表现差，对应**欠拟合**。
- **屋顶工 2**：1 毫升（漏得很少）→ 表现好，对应**合适拟合**。
- **屋顶工 3**：0 毫升（完全不漏）→ 单看这个数字「最好」，但结合上文类比，其做法过度复杂，对应**过拟合**（在真实场景中可能对应对训练数据过度贴合、泛化变差）。

**若只看工作表现**：屋顶工 1 很糟糕（屋顶还在漏水）；屋顶工 3 可能看起来「更好」（修到完全不漏）。但**仅凭表现指标不够**——会错误地选屋顶工 3 而非屋顶工 2。因此还要引入**复杂度**的度量。

---

## 复杂度指标：修理收费（美元）

用**修理屋顶的收费（美元）**来衡量方案的「复杂度」或成本：

- **屋顶工 1**：1 美元（极低成本，但修不好）→ 复杂度很低。
- **屋顶工 2**：100 美元（适中）→ 复杂度适中。
- **屋顶工 3**：100 000 美元（极高）→ 复杂度很高。

---

## 综合指标：工作表现 + 复杂度

把**工作表现**（如漏水量）和**复杂度**（如价格）**结合起来**，才能做正确选择。例如将「漏水量」与「价格」相加（或按某种权重组合），得到综合分：

- **屋顶工 1**：1000（漏水量）+ 1（美元）→ **1001**
- **屋顶工 2**：1（漏水量）+ 100（美元）→ **101**
- **屋顶工 3**：0（漏水量）+ 100 000（美元）→ **100 000**

综合后**屋顶工 2 的分数最低**，显然最好。这表示：**同时优化性能和复杂度**，会得到**尽可能简单又足够好**的结果。

---

## 另一个过拟合的例子：电影推荐

过拟合还有一种更**隐蔽**的形式：与多项式阶数无关，而与**特征数量**以及**系数的大小**有关。

**场景**：某电影流媒体平台有 10 部已有电影（M1～M10），新上线电影 M11。希望根据用户对这 10 部电影的观看时长，用线性回归预测用户会看 M11 多久，从而做推荐。数据集共 100 个用户。**特征** x₁～x₁₀：用户观看 M1～M10 的时长（秒），未观看则为 0。**标签**：用户观看 M11 的时长。

**模型**：ŷ = w₁x₁ + w₂x₂ + … + w₁₀x₁₀ + b，其中 ŷ 为预测的 M11 观看时长，wᵢ 为对应电影的权重，b 为偏差。

**两个模型的对比**：

- **模型 1**：只有少数权重非零（例如主要与电影 3、7、9 相关），系数数值不大。模型简单、可解释：例如电影 3、7 的系数为正，表示看过 M3 或 M7 的用户较可能看 M11；电影 9 的系数为负，表示看过 M9 的用户较不可能看 M11。多数系数为 0，说明模型只用了少量信息，更像在**学规律**。
- **模型 2**：10 个特征都有较大系数，且数值很大、正负交错（如 22x₁ − 103x₂ − 14x₃ + 109x₄ − 93x₅ + …）。直觉上很难相信「用户看 M2 的时长要乘以 −103 再加一堆数」能代表真实规律；更像是**把训练数据背下来了**，而不是学到可泛化的结构。因此**模型 2 更像过拟合**。

小结：过拟合不仅可能来自**模型形式**太复杂（如高阶多项式），也可能来自**系数过多、数值过大**。正则化通过惩罚权重大小（及非零个数），可以抑制这类过拟合，使模型更接近「模型 1」式的简单、可解释形式。

---

## 衡量模型复杂度：L1 范数与 L2 范数

要为「复杂度」定一个可算的指标，才能把它写进损失函数做正则化。常用两种方式：**系数绝对值之和**（L1 范数）和**系数平方和**（L2 范数）。这样可以把「模型 1 较简单、模型 2 较复杂」用公式体现出来：**系数越多或数值越大**，模型往往越复杂。

**定义**：
- **L1 范数**：所有权重（系数）的**绝对值之和**。
- **L2 范数**：所有权重的**平方和**。

二者来源于一般的 Lᵖ 范数理论（以法国数学家勒贝格命名）。为什么要用绝对值或平方？因为若直接相加，很大的正系数和很大的负系数会相互抵消，反而得到很小的数，无法反映模型真正的复杂度；取绝对值或平方后，正负都贡献为「大」的复杂度。

**计算时的一个约定**：**偏差（常数项 b）不参与 L1、L2 的计算**。偏差表示「所有特征为 0 时的预测值」（如用户前 10 部电影都没看时对 M11 的预测），与模型「用了多少特征、系数多大」的复杂度无关，因此只对**权重**做范数，不把 b 算进去。

**示例（沿用电影推荐中的模型 1 与模型 2）**：

- **模型 1**：有效权重例如 2、1.4、−0.5（对应 x₃、x₇ 等），偏差 8 不参与。
  - L1 = |2| + |1.4| + |−0.5| = **3.9**
  - L2 = 2² + 1.4² + (−0.5)² = **6.21**
- **模型 2**：10 个权重都较大（如 22, −103, −14, 109, …），偏差 8 不参与。
  - L1 = |22| + |−103| + … = **1089**
  - L2 = 22² + (−103)² + … = **227131**

正如预期，**模型 2 的 L1、L2 都远大于模型 1**，说明模型 2 复杂度更高。对多项式回归也可以同样处理：对多项式系数（不含常数项）求 L1 或 L2，作为复杂度的度量，再在损失函数里加上这一项（或它的倍数），就得到** L1 正则化**和** L2 正则化**（见下文 Lasso / 岭回归）。

## 修改误差函数来解决问题：Lasso 回归与岭回归

把「性能」和「复杂度」一起放进要最小化的目标，就得到**修改后的误差函数**：用**性能指标**（即原来的误差函数）加上**复杂度指标**（L1 或 L2 范数），训练时最小化二者之和。屋顶工类比：要找「修得好且不太贵」的屋顶工，等价于让「漏水量 + 价格」尽量小；对应到模型，就是让 **回归误差 + 正则化项** 尽量小。

**两项分别是什么**：
- **回归误差**：衡量模型拟合好坏，即第 3 章学过的**绝对误差**或**平方误差**（在训练集上的平均等）。
- **正则化项**：衡量模型复杂度，即权重的 **L1 范数**或 **L2 范数**（偏差不参与）。

因此：**要最小化的误差 = 回归误差 + 正则化项**。实际使用时还会在正则化项前乘一个系数（正则化强度），用来调节「拟合」与「简单」的权衡。

**按使用的范数命名**：
- **Lasso 回归（Lasso regression）**：用 **L1 范数**做正则化。Lasso 即「**最小绝对收缩和选择算子**」（Least Absolute Shrinkage and Selection Operator）。  
  **Lasso 的误差函数 = 回归误差 + L1 范数**（或加上系数后的倍数）。
- **岭回归（Ridge regression）**：用 **L2 范数**做正则化。「岭」的由来：在回归误差上加上 L2 项后，误差函数的图像中原本尖锐的峰会被「磨平」成平滑的谷，形状像山岭。  
  **岭回归的误差函数 = 回归误差 + L2 范数**（或加上系数后的倍数）。

Lasso 与岭回归在**实践中都常用、效果良好**；二者在「稀疏性」「稳定性」等方面各有特点，选择哪一种往往取决于具体偏好与任务（见下文的系数影响）。在具体实现时，需要引入**正则化参数** λ，并用验证集选择其取值。

## 在模型中调节性能与复杂度：正则化参数 λ

训练目标是让**总误差**尽量小，即同时兼顾「表现好」和「不太复杂」。但二者有**权衡**：一味提高表现可能让模型变复杂，一味压低复杂度可能让表现变差。大多数机器学习方法都有可调的**超参数**，用来在两者之间取得平衡；正则化中的这个超参数就是**正则化参数**，通常记作 **λ（lambda）**。

**作用**：λ 决定训练时更看重「性能」还是「简单」。做法是把正则化项乘以 λ 后加到回归误差上，用得到的**总误差**来训练模型：

**误差 = 回归误差 + λ × 正则化项**

- **λ = 0**：正则化项被消掉，等价于第 3 章的不带正则化的普通回归。
- **λ 很大**：模型会变得很「简单」（如系数普遍变小或很多变为 0），可能对数据拟合不足。
- **选合适的 λ 很重要**；**验证集**是常用手段：对多个 λ（如 10、1、0.1、0.01 等 10 的幂）分别训练模型，在**验证集**上比较表现，选验证误差最小的那个 λ（及对应模型）。

## L1 与 L2 正则化对模型系数的影响

L1（Lasso）和 L2（岭）表面形式相近，但对**系数**的影响不同，选用哪一种会直接影响得到的模型形态。

仍以**电影推荐**为例：初始模型可能是 ŷ = 22x₁ − 103x₂ − 14x₃ + 109x₄ − 93x₅ + 203x₆ + 87x₇ − 55x₈ + 378x₉ − 25x₁₀ + 8（系数多且大）。加入正则化并重新训练后，会得到更简单的模型。

- **L1 正则化（Lasso）**：倾向于让**部分系数变为 0**，即得到**更少非零系数**的模型（**稀疏**）。例如训练后可能变成 ŷ = 2x₃ + 1.4x₇ − 0.5x₉ + 8，只有少数电影（如 M3、M7、M9）的权重非零，其余为 0，便于解释和特征选择。
- **L2 正则化（岭回归）**：倾向于让**所有系数整体变小**，但**很少把系数精确压成 0**。即系数会「收缩」、数值变小，但大多仍非零。

因此：若希望模型**稀疏、便于解释或做特征选择**，可优先考虑 **Lasso（L1）**；若主要希望**抑制系数过大、稳定拟合**，可考虑**岭回归（L2）**。

**何时用 L1、何时用 L2**：**L1** 更适合**特征很多**、且希望**去掉其中一大部分**（让对应系数为 0）的情形；**L2** 更适合**特征不多**、且认为**这些特征都有用**、不想把有用特征「删掉」的情形。

- **电影推荐**（见前文「另一个过拟合的例子」）：每个特征对应一部电影，目标是从多部电影里找出**少数**与用户兴趣相关的，因此需要**多数系数为 0、少数非零**的模型 → **L1 更合适**。
- **多项式回归**（如 01 中 4.1 的多项式例子）：若只有**一个特征 x**，L2 正则化能帮助得到**系数较小**的多项式，曲线振荡不那么剧烈，从而减轻过拟合；第 4.6 节会演示多项式例子中 L2 是合适选择。

**深入理解**：为何 L1 容易把系数压成 0、L2 容易把系数压成小分数，可从数学上在**附录 C** 中查看。下一节将介绍**直观理解正则化的方式**，用例子说明 L1 与 L2 在惩罚复杂度上的差异；若要看带正则化的梯度下降等数学形式，可参考**附录 B**（用梯度下降做正则化）。

**从「误差」到「如何改进」**：理解机器学习模型不能只看「误差函数说把误差减小就能得到好模型」，这有点像只说「人生少犯错就能成功」。更好的理解是**正面**的：**可以做哪些事来改善**。第 3 章用「绝对技巧」「平方技巧」一步步把线往数据点靠近，重复多轮得到好的线性拟合；类似地，理解正则化时也可以结合第 3 章线性回归算法的每一步，看正则项如何在这一过程中起作用（更具体的分析可重复第 3 章定义的步骤）。

## 直观理解正则化的方式：在每一步「缩小系数」

**线性回归算法的伪代码**（与第 3 章一致）：**输入**点数据集，**输出**适合该数据集的线性回归模型。**步骤**：先选一个带随机权重和随机偏差的模型；然后**重复多次**：选一个随机数据点，**稍微调整权重和偏差**以改进对该点的预测；最后得到训练好的模型。

**能用同样的思路理解正则化吗？可以。** 为简单起见，假设训练时我们还想让**模型更简单**，做法之一就是**把系数缩小一点**。例如模型有 3 个系数：3、10、18。能不能把这三个数都稍微减小？可以，且有两种常见实现方式。取一个小数 **λ**（如 λ = 0.01），在算法的每个阶段对系数做一次「缩小」：

- **方法一**：对每个**正**系数**减去 λ**，对每个**负**系数**加上 λ**；若系数为 0 则不动。  
  例：(3, 10, 18) → (2.99, 9.99, 17.99)。  
  这种「向 0 靠拢一步」的更新，与 **L1 正则化**的直观效果类似（长期会有一部分系数被压成 0）。
- **方法二**：把所有系数**乘以 (1 − λ)**。因 λ 很小，(1 − λ) 接近 1。  
  例：(3, 10, 18) × 0.99 → (2.97, 9.9, 17.82)。  
  这种「按比例缩小」与 **L2 正则化**的直观效果类似（系数整体变小，但很少精确为 0）。

两种方法都在**缩小系数**。λ 在这里很像**学习率**，与正则化强度（正则化率）密切相关，数学上在**附录 B**（用梯度下降做正则化）中有详细形式。

**与 L1/L2 的对应关系**：若在每一步用**方法一**做缩小，就相当于在做 **L1 正则化（Lasso）** 训练；若用**方法二**，就相当于在做 **L2 正则化（岭回归）** 训练。前文已说明 L1 容易把很多系数压成**恰好 0**，L2 则多是把系数**变小**而很少精确为 0；下面用数值例子把这一差异看得更清楚。

**带正则化的训练流程**：**输入**点数据集，**输出**适合该数据集的线性回归模型。**步骤**：(1) 选一个带随机权重和随机偏差的模型；(2) **重复多次**：选一个随机数据点 → 稍微调整权重和偏差以改进对该点的预测 → **用方法一或方法二稍微缩小系数**；(3) 得到训练好的模型。

**L1 把系数压成 0 的数值例子**：设某系数初始为 **2**，正则化参数 **λ = 0.01**。若在每一轮都用**方法一（L1）**缩小：正系数减 λ，则 2 → 1.99 → 1.98 → … → 0.02 → 0.01 → **0**。一旦到 0，规则是「若为 0 则不动」，因此之后一直保持 0。例如重复约 200 次后，该系数变为 0 且不再变化。这就是 L1 能产生**稀疏**（很多系数为 0）的直观原因。

**L2 的对比**：若用**方法二（L2）**，每步把系数乘以 (1 − λ)。同样从 2 开始、λ = 0.01，重复 200 次后系数变为 2 × (0.99)^200，是一个很小的正数，但**不会精确等于 0**。因此 L2 会让系数整体变小，却很少把系数压成恰好 0。

---

## 正则化的意义

正则化的含义正是：用**两个不同的误差（或惩罚）**分别衡量**性能**（如预测误差）和**复杂度**（如权重的 L1/L2 范数），在目标函数里同时最小化二者，从而在单次训练中避免过拟合、找到性能与复杂度的平衡。L1、L2 范数就是常用的复杂度度量。
