# 第 2 章笔记：机器学习类型

## 一、机器学习的本质与应用

### 本质

- 机器学习的核心是：**基于历史数据做决策**。  
- 它模拟了人类“从经验中学习”的过程，不依赖手工写死每条规则。  

### 典型应用场景

- 预测类：房价预测、股票价格趋势分析  
- 风险识别：垃圾邮件检测、信用卡欺诈检测  
- 感知识别：人脸识别、动物图像识别  
- 文本与推荐：文本摘要、内容推荐、聊天机器人  
- 智能决策：自动驾驶、疾病诊断、用户分组、棋类博弈  

## 二、三大主要分支

根据学习方式，机器学习通常分为三类：

1. **监督学习（Supervised Learning）**  
   使用带标签的数据训练模型，用于分类或回归预测。  
   （本书当前阶段重点）

2. **无监督学习（Unsupervised Learning）**  
   使用无标签数据，挖掘隐藏结构与模式（如聚类、降维）。

3. **强化学习（Reinforcement Learning）**  
   智能体通过与环境交互，依据奖励信号学习最优策略。

## 三、数据、特征与标签

### 什么是数据

- 数据本质上是信息的记录。  
- 在表格数据里，每一行是一个样本，每一列是该样本的某个属性。  

### 什么是特征

- 特征是描述样本属性的字段（列），例如大小、名称、重量、颜色等。  
- 特征用于表达“已知信息”，是模型做判断的输入。  

### 什么是标签

- 标签是任务希望预测的目标，具体含义取决于问题场景。  
- 同一批特征，在不同任务里可以对应不同标签：  
  - 预测宠物类别 -> 标签是“猫/狗”；  
  - 预测宠物健康状态 -> 标签是“健康/生病”；  
  - 预测年龄 -> 标签是一个数值。  

### 什么是预测

- 预测是模型根据特征对标签做出的猜测。  
- 机器学习训练的目标，就是让这种猜测尽可能接近真实标签。  

### 标签数据 vs 无标签数据

- **标签数据（Labeled Data）**：样本同时包含特征和已知标签，常用于监督学习。  
- **无标签数据（Unlabeled Data）**：样本只有特征，没有目标标签，常用于无监督学习。  

## 四、监督学习速记（精简版）

- **输入**：特征（X）  
- **输出**：标签（y）  
- **目标**：学习映射关系 `X -> y`，让模型对新样本也能正确预测。  
## 一句话总结

> 第 2 章的主线是：先明确机器学习的三种类型，再把“数据-特征-标签”这套语言体系建立起来，为后续监督学习打基础。

## 2.1 补充示例：标签数据和无标签数据

### 图中示例在说明什么

- 标签数据 = 每个样本都带有目标答案。  
- 无标签数据 = 只有样本特征，没有目标答案。  

### 宠物数据的直观理解

- **标签数据示例 1**：宠物图片 + 类别（如“狗/猫”）。  
- **标签数据示例 2**：宠物图片 + 体重（如“18 磅/9 磅”）。  
- **无标签数据示例**：只有宠物图片，没有类别或体重等目标列。  

### 为什么同一批数据有时“有标签”，有时“无标签”

- 是否算“标签数据”，取决于你当前要解决的问题。  
- 如果任务是“识别猫狗”，那么“类别”就是标签。  
- 如果任务是“预测体重”，那么“体重”就是标签。  
- 如果没有提供你要预测的目标列，这批数据对该任务就是无标签数据。  

### 对应学习方式

- 有标签数据通常用于监督学习。  
- 无标签数据通常用于无监督学习。  

### 一句话总结

> 标签不是数据天生自带的属性，而是由“你要预测什么”这个任务目标决定的。

## 2.2 监督学习：处理有标签数据的分支

### 核心定义

- 监督学习使用**有标签数据**训练模型。  
- 目标是学习“特征 -> 标签”的映射关系，然后对新样本进行预测。  

### 工作流程（记忆-制定-预测）

- **记忆**：读取带标签训练数据（样本 + 正确答案）。  
- **制定**：通过训练算法找到最贴合数据的规则/模型。  
- **预测**：输入新样本特征，输出其最可能的标签。  

### 监督学习常见任务类型

#### 1) 回归（Regression）

- 标签是连续数值（数值型数据），如体重、房价、气温。  
- 模型输出一个具体数值。  

#### 2) 分类（Classification）

- 标签是离散类别（类别型数据），如猫/狗、垃圾邮件/非垃圾邮件。  
- 模型输出一个类别或类别概率。  

### 结合图中示例理解

- 当标签是“体重”时，任务属于回归。  
- 当标签是“宠物类别”时，任务属于分类。  
- 同一组输入特征，因标签不同，会形成不同监督学习任务。  

### 本节要点

- 监督学习的前提是训练集必须有目标标签。  
- “标签类型”决定任务类型：连续值 -> 回归，离散类 -> 分类。  
- 监督学习模型本质上是在已知答案样本上学习，再推广到未知样本。  

### 一句话总结

> 监督学习就是用“带答案的数据”训练模型，其中预测数值叫回归，预测类别叫分类。

## 2.2 补充：回归与分类的典型任务

### 示例 1：房价预测（回归）

- 在房价任务中，每个样本是一套房。  
- 输入特征可以包括：面积、卧室数、地段、房龄等。  
- 标签是连续数值（房屋价格），所以这是回归问题。  
- 模型输出通常是一个具体价格区间内的估计值。  

### 示例 2：垃圾邮件检测（分类）

- 在邮件任务中，每个样本是一封邮件。  
- 输入特征可包括：邮件长度、关键词、是否有附件、发件人信息等。  
- 标签是离散类别（垃圾邮件/非垃圾邮件），所以这是分类问题。  
- 模型输出是类别或对应概率。  

### 回归任务常见场景

- 房屋价格预测  
- 医疗费用或保险赔付金额预测  
- 销售额、需求量、能耗等连续数值预测  
- 股票或其他时间序列的数值趋势估计  

### 分类任务常见场景

- 图像分类（如猫狗识别）  
- 垃圾邮件识别  
- 欺诈检测（正常/异常）  
- 医疗初筛（某疾病风险高/低）  
- 用户行为分类（是否流失、是否点击等）  

### 本节要点

- 区分回归和分类的关键，不是输入特征，而是**标签类型**。  
- 标签是数值 -> 回归；标签是类别 -> 分类。  
- 同一业务可以拆成不同任务，因此也可能同时包含回归与分类。  

### 一句话总结

> 监督学习里，“预测多少”是回归，“预测属于哪一类”是分类。

## 2.3 无监督学习：处理无标签数据的分支

### 核心定义

- 无监督学习处理的是**无标签数据**。  
- 与监督学习不同，它没有“标准答案”可直接对照。  
- 目标是从数据本身发现结构、规律和潜在关系。  

### 这类方法在做什么

- 当没有标签时，模型会尝试回答：  
  - 哪些样本彼此更相似？  
  - 是否可以分成若干组？  
  - 数据是否存在异常点或隐藏模式？  

### 常见无监督学习任务

#### 1) 聚类（Clustering）

- 把相似样本自动分为一组。  
- 例如在邮件场景中，不依赖“垃圾/非垃圾”标签，也可按词频、附件大小、发件时间等特征把邮件分群。  

#### 2) 降维（Dimensionality Reduction）

- 在尽量保留关键信息的前提下，压缩特征维度。  
- 常用于可视化、降噪和后续建模加速。  

#### 3) 生成建模（Generative Modeling）

- 学习数据分布并生成与原数据相似的新样本。  
- 常见于图像、文本等数据生成任务。  

### 结合图中直观理解

- 图里把相似动物放到一起，本质就是“按相似性分组”的思想。  
- 这种分组不依赖人工先给标签，属于典型无监督学习过程。  

### 本节要点

- 无监督学习不是“预测已知答案”，而是“发现数据内部结构”。  
- 在真实业务里，它常用于探索性分析、用户分群、异常检测和特征工程。  
- 监督学习和无监督学习可以结合使用：先无监督找结构，再监督做预测。  

### 一句话总结

> 无监督学习是在“没有标准答案”的情况下，让模型先看懂数据的组织方式。
